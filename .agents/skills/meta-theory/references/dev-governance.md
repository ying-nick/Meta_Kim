# 开发治理流程 — 完整参考

## 完整流程

```
[接收任务]
  ↓
1. Critical: 需求分析
   - 需求清晰度检测
   - 抽象化为"需要什么类型的元"
   - 复杂度评估
   - 风险评估
   ↓
2. Fetch: 搜索获取
   - 搜索现有agents（.claude/agents/*.md）
   - 搜索skills（.claude/skills/*/SKILL.md）
   - 验证能力匹配度
   ↓
3. Critical: 决策
   IF 找到匹配 → 验证边界 → 选择agent
   IF 未找到 → 触发创建流水线（类型B）
   ↓
4. Execution: 执行
   - 任务分配
   - 文件所有权声明
   - 并行/串行决策
   ↓
5. Review: 评审
   - 代码质量评审（代码审查能力）
   - UX体验评审（体验审查能力）
   - 安全评审（安全审查能力）
   ↓
6. Meta-Review: 元评审
   - agent越界检测（meta-prism）
   - 架构合规性检测
   - KISS/DRY/SOLID检测
   ↓
7. Revision: 修订（如需）
   - 根据评审意见修订
   - 重新提交Review
   - 最多2轮
   ↓
8. Evolution: 意图放大（详见下方5维度展开）
   ↓
[输出结果 + 进化建议]
```

## 复杂度路由

```
[复杂度评估]
  ├─ 简单（<2文件改动）→ 直接执行 → Review
  ├─ 中等（2-5文件）→ Fetch → Review → MetaReview
  └─ 复杂（>5文件/多模块）→ 完整流程
```

## 元技能调用

| 阶段 | 元技能 | 具体调用 |
|------|--------|---------|
| Critical分析 | 批判性分析能力 | 需求清晰度、复杂度、风险评估 |
| Fetch搜索 | 仓库搜索能力 | 搜索agents + skills |
| Fetch搜索 | 外部技能搜索能力 | 搜索skills.sh生态 |
| Review评审 | 代码审查能力 | 代码质量 |
| Review评审 | 体验审查能力 | UX体验 |
| Review评审 | 安全审查能力 | 安全扫描 |
| Meta-Review | meta-prism | 越界检测 + AI-Slop |
| Evolution | 意图放大检查 | 5维度检测 |

## Evolution: 5维度进化检测（展开）

任务完成后必须执行。核心思想：**同一意图，不该只有一种表达方式。成熟系统按触点、角色、上下文换壳不换核。**

### 维度1：模式复用检测

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 解法可抽象性 | 这次的实现里有没有可复用的模式/组件/模板？ | 有 → 提取为通用模块或skill |
| 重复代码 | 跨文件/跨agent有没有类似逻辑？ | 有 → DRY重构或抽象为共享工具 |
| 配置模板化 | 有没有硬编码可以变成配置？ | 有 → 提取为模板 |

### 维度2：agent边界检测

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 5标准仍成立 | 重新跑一遍5标准验证表 | 5/5 PASS |
| 职责漂移 | 这次任务中agent有没有"顺手"做了不该做的事？ | 无越界行为 |
| 拆分/合并信号 | 有没有agent太胖（>2域）或太碎（无法独立产出）？ | 无死法信号 |

### 维度3：引导优化检测

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 追问轮数 | 从用户输入到开始执行，追问了几轮？ | ≤2轮为优，>3轮需优化引导 |
| 输入简化 | 用户是否需要提供过多上下文才能启动？ | 能从历史/记忆中自动补全 |
| 误解率 | 有没有因误解需求导致返工？ | 0次为优 |

### 维度4：流程瓶颈检测

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 最慢环节 | 哪一步耗时最长或卡住最久？ | 已识别并有改进方案 |
| 失败热点 | 哪一步最容易出错/需要重来？ | 已加防护或前置检查 |
| 串行浪费 | 有没有本可并行但被串行执行的步骤？ | 已标记可并行 |

### 维度5：能力覆盖检测

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 新能力缺口 | 这次任务暴露了什么能力缺失？ | 缺口已记录 |
| 工具/Skill缺失 | 有没有需要但不存在的tool/skill？ | 已触发创建流水线或记录需求 |
| 知识缺失 | 有没有需要但不知道的领域知识？ | 已添加到记忆/references |

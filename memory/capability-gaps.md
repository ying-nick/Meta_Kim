# Capability Gaps

This file records durable capability gaps discovered during Evolution.

## Schema

- `date`: When the gap was observed.
- `context`: Which run, task, or trigger exposed the gap.
- `gap`: Missing capability, owner, tool, or protocol.
- `impact`: Why the gap matters.
- `proposed_owner`: Existing owner, new owner, or temporary fallback.
- `status`: `open`, `in_progress`, `resolved`, or `retired`.
- `resolution`: Link to the asset or change that closed the gap.

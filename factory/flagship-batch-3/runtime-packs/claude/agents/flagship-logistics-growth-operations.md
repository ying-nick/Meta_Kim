## Activate When

- delivery failures are rising and nobody agrees where the network is breaking
- service levels are unstable across nodes
- route and node metrics exist but there is no operating system around them
- recovery after misses is chaotic and trust is degrading

## Decision Rules

- If the node-level fix harms the network, reject it.
- If service-level tradeoffs are hidden, surface them before any optimization call.
- If the route problem is actually an inventory or handoff problem, say so early.
- If recovery loops depend on heroic manual effort, redesign the cadence.
- If cost wins come from invisible service damage, call that out directly.
